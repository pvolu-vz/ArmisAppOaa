# OAA Connector Test-and-Fix Loop

Generic, introspection-driven pytest harness for any connector under
`./integrations/<slug>/`. Catches the classes of bugs that only surface at
dry-run time today:

- Importing a class that doesn't exist (`CustomHRISProvider` vs `HRISProvider`)
- Calling a method that doesn't exist (`app.add_permission(...)`)
- Passing a kwarg the SDK doesn't accept (`add_resource(type=...)` instead of `resource_type=`)
- Referencing an enum attr that doesn't exist (`OAAPermission.WriteData` vs `DataWrite`)
- Reading a CSV column that isn't present in `samples/*.csv`

## How it works

```
   ┌──────────────────────┐    ┌──────────────────────┐    ┌────────────────┐
   │ introspect.py        │    │ analyze.py           │    │ generate.py    │
   │  oaaclient.client    │───▶│  AST walk of         │───▶│  emits pytest  │
   │  oaaclient.templates │    │  integrations/<slug> │    │  test_*.py     │
   │  oaaclient.structures│    │  /<slug>.py          │    │                │
   └──────────────────────┘    └──────────────────────┘    └────────────────┘
                                                                    │
   ┌──────────────────────┐    ┌──────────────────────┐             ▼
   │ run_loop.sh          │◀───│ harness.py           │◀── runs pytest
   │  iter counter +      │    │  emits structured    │
   │  dry-run check       │    │  .test_report.json   │
   └──────────────────────┘    └──────────────────────┘
```

No code is mocked. The tests assert against the **real installed `oaaclient`
package** — when oaaclient bumps a method signature, regenerated tests catch
the regression automatically.

## Generated test modules

For `integrations/<slug>/` the harness writes these files into
`integrations/<slug>/tests/`:

| File | What it asserts |
|---|---|
| `test_import.py` | Connector module imports without raising — catches `ImportError`, `SyntaxError`, missing deps |
| `test_sdk_signatures.py` | Every `<oaa_class>.<method>(...)` call site uses an existing method, a valid kwarg set, and a legal positional arg count |
| `test_sdk_enums.py` | Every `OAAPermission.X` / `OAAIdentityType.X` / `IdPProviderType.X` reference exists on the SDK class |
| `test_schema_contract.py` | Every CSV column referenced as `row["X"]` / `row.get("X")` exists in at least one file under `samples/` |
| `conftest.py` | Puts the connector directory on `sys.path` |
| `_inventory.json` | Debug dump of the AST walk's call-site inventory |

All five files carry an `AUTO-GENERATED by tools/oaa_test_loop` header.
The harness re-generates them every run; hand edits are clobbered.

## Usage

### One-shot run

```bash
./tools/oaa_test_loop/run_loop.sh <slug>
```

This regenerates the test suite, runs `pytest`, and — if green — invokes the
connector with `--dry-run --save-json`. Exit code 0 only if both pass.

### Autonomous loop (Claude or human-driven)

The loop is **stateless** between iterations — each invocation reads
`./integrations/<slug>/.test_loop_state.json` for the iteration counter and
bumps it. After each failing iteration:

1. Read `./integrations/<slug>/.test_report.json` — structured JSON with the
   per-test failure messages.
2. Edit `integrations/<slug>/<slug>.py` to fix the root cause.
3. Re-invoke `./tools/oaa_test_loop/run_loop.sh <slug>`.

The state file aborts after `--max-iter` (default 15) without going green.

### Just the harness (no dry-run)

```bash
python3 -m tools.oaa_test_loop <slug>
# or:
python3 -m tools.oaa_test_loop <slug> --no-regen          # skip regen
python3 -m tools.oaa_test_loop <slug> --venv venv/bin/python  # specific venv
python3 -m tools.oaa_test_loop <slug> --pytest-args "-k smoke"
```

The harness writes `integrations/<slug>/.test_report.json` with:

```json
{
  "slug": "<slug>",
  "exit_code": 0,
  "passed": true,
  "summary": "15 passed in 0.08s",
  "regeneration": { "spec_classes": 40, "spec_methods": 193, "calls_traced": 7, ... },
  "failures": [ { "nodeid": "...", "message": "..." } ],
  "stdout_tail": "...",
  "stderr_tail": "..."
}
```

## Sanity check / regression fixture

`integrations/demo_broken/` is a known-good fixture. It originally contained
five deliberate bugs (see the connector docstring). Running the loop against
it should produce 15 passed and a clean dry-run:

```bash
./tools/oaa_test_loop/run_loop.sh demo_broken
```

If this ever fails after a change to the framework, the AST analyzer or test
generator has regressed.

## Limitations

- The AST analyzer is **best-effort** — when a method call's receiver class
  can't be traced (e.g. through a function return that isn't on the heuristic
  table in `analyze.py`), the call is recorded in `unresolved_calls` and
  skipped for signature checks. Add to `_guess_return_type()` if a common
  pattern is missed.
- Schema-contract tests only check columns referenced via the receivers
  `row`, `r`, `record`, `rec`, `rows` — the conventional CSV-iteration names.
  References through other variable names are ignored.
- The dry-run step assumes the connector accepts `--data-dir`, `--dry-run`,
  `--save-json`, `--log-level` (the CLI contract from CLAUDE.md). Connectors
  that diverge from that contract will fail the dry-run step.
